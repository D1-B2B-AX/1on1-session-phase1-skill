# AX 1on1 세션 Phase 1 — 형식지 추출 스킬

> ※ 본문 초안(draft). 실제 위치 = `~/.claude/skills/형식지추출/SKILL.md`.

## §1. 역할

- 1:1 세션 신청자의 *원업무*를 형식지로 복원하기 위해, 인터뷰를 **[전] 준비**하고 **[후] 정리**한다.
- **인터뷰 진행 자체는 사람(AX 매니저)이 한다 — 스킬은 인터뷰를 대신하지 않는다.**
- 두 단계는 **무상태**: 각 단계는 *세션 폴더*에서 읽고 쓴다. 이전 실행 기억에 의존하지 않는다.
- 원칙: 고정 템플릿(11칸) + input만큼 맞춤 질문 · 유도 금지(열린 질문) · 구현 방향 단정 금지(= Phase 2 영역).

## §2. 진입 — 자동 업데이트 · 신청자·업무 확인 · 단계 감지

`/형식지추출` 실행 시:
1. **자동 업데이트**: 스킬 폴더(`~/.claude/skills/형식지추출/`)에서 `git pull --quiet`. 네트워크 오류면 무시하고 계속.
2. **★ 신청자·업무 먼저 받기 (필수 — 단계 감지의 전제)**: "어느 신청자(이름)의 어떤 업무 세션인가요?"를 *반드시 먼저* 묻는다.
   - 이유: [전] 단계엔 아직 세션 폴더가 없다. 폴더 내용(raw 유무)만으로는 단계를 알 수 없고, 애초에 *어느 폴더를 봐야 하는지*도 모른다. 신청자·업무를 알아야 해당 세션 폴더가 있는지·관련 파일(구글폼응답·raw)이 있는지 확인할 수 있다. [후]도 동일 — 신청자 입력과 연결된 폴더를 찾으려면 필요.
3. **세션 폴더 조회·확정**: 받은 신청자·업무로 `~/Desktop/AX-1on1-sessions/{신청자}_{업무}_세션/`을 찾는다. (바탕화면 경로는 실행 시점 탐색 — OneDrive면 `~/OneDrive/Desktop`. 절대경로 기록.)
   - 폴더 없음 → 새 세션 → 폴더 생성하고 [전] 추정.
   - 폴더 있음 · 인터뷰 raw 없음 → [전] 추정.
   - 폴더 있음 · raw 있음 → [후] 추정.
4. **단계 확인** (조용히 분기 X): 추정 단계를 사용자에게 확인 — "[전: 인터뷰질문 만들기] / [후: 형식지 작성] 맞나요?" 확인 후 §3 / §4. raw가 다른 폴더에 있으면 사용자가 "[후] 단계, raw는 이 경로"라고 알려주면 그대로 §4.

## §3. [전] — 세션별 인터뷰질문 생성

**reads**: `references/형식지템플릿.md`(11칸 슬롯) · `references/인터뷰프로토콜.md`(질문 은행·라이브 원칙) · `references/구글폼슬롯매핑.md`(매핑·보강 규칙) · `references/인터뷰질문양식.md`(출력 양식 = 콘솔 v0)

1. **input 요청**: "구글폼 응답을 주세요 — 응답 스프레드시트 *다운로드 파일*이나 해당 신청자 응답. 첨부(양식 URL·파일) 있으면 같이." (링크 직접 읽기는 MCP 필요 → 기본은 다운로드 파일.)
2. **신청자 행 추출**: 전체 시트면 **이메일(Q3)로 매칭**한 행만 추출 / 신청자 응답만 붙여넣었으면 그대로 사용 / 매칭 실패 시 사용자에게 확인. → `구글폼응답_{신청자}.md`로 세션 폴더에 저장(스냅샷). 다른 신청자 행은 남기지 않음(개인정보). 첨부는 폴더에 따로.
3. **인터뷰질문 생성** (`구글폼슬롯매핑.md` 3단계 적용). **★ 구글폼에 없는 정보(분야·세부 등)를 질문·맥락에 주입하지 않는다(지어내기 금지) — 단정 말고 *물어라*. 이 원칙은 보강뿐 아니라 [전] 생성 전체에 적용.**
   - ① 구글폼을 11칸에 pool → 칸별 **충분/부족/빔** 표시
   - ② 구글폼이 못 주는 깊이(**2 시작완료·1 수혜자/목적·1 전체/일부 담당범위·5 판단기준·7 통과기준·8 예외**)는 표준 질문 *무조건* 포함 (질문 본문은 인터뷰프로토콜에서)
   - ③ **신호 OR 부족** 칸에 보강 질문 (구글폼슬롯매핑 §5 앵커대로 · *적게·열린 형태*)
4. **output** = `인터뷰질문_{신청자}_{업무}.md` (세션 폴더). `인터뷰질문양식.md`(=콘솔 v0)의 양식(코드블록 질문 흐름·`├└` 분기·진행 마커)을 따른다. **단일 자급자족 파일** — AX 매니저가 이 1개만 보고 인터뷰하므로 아래를 *전부* 담는다:
   - 질문 흐름 (충분 칸=확인·압축 / 부족·빔·깊이 칸=필수 / 보강 질문)
   - *어떻게 묻나(HOW)*: 꼬리질문·모호어 캐묻기·막힘 분기·침묵 (인터뷰프로토콜 라이브 원칙)
   - 진행 순서·넘어가기 기준
   - 끝부분에 **페인·개선 희망 질문** (인터뷰프로토콜 7.5단계)
5. **마무리 안내**: "이 `인터뷰질문_...md`를 보며 인터뷰 진행하세요. 끝나면 녹음/메모(raw)를 이 폴더에 넣고 다시 `/형식지추출`를 실행하면 형식지를 만들어드려요."

## §4. [후] — 형식지 작성

**reads**: `references/형식지템플릿.md` · `references/후처리규칙.md`

1. **input**: 인터뷰 raw + `구글폼응답_{신청자}.md` + 첨부.
   - raw 수령 3-way: ① 폴더 파일 ② 경로 붙여넣기(*타 폴더 OK*) ③ 텍스트 붙여넣기. **없으면 리마인드**: "이 폴더에 raw를 넣거나 경로/내용을 알려주세요."
2. **작성**: `후처리규칙.md`의 규칙 1~9 그대로 적용 — raw+구글폼을 11칸 매핑 / 추론은 "(추론)" / 못 채운 건 10번 / 구현 신호는 11번 / 권한 `[권한·정책]` 태그 / 암묵지 라벨 절제 / 8번 빈칸 라벨.
3. **output**:
   - `Phase1_형식지_{신청자}_{업무}.md` (세션 폴더) — 11칸(**10번 미확인 포함**) + 끝에 **★ 작성자 검토 필요 목록**.
   - **대화창 echo**: ★ 검토 목록 + 10번 미확인 요약을 출력 + **"이 내용은 형식지 파일 안에도 있어요"** 안내.
   - **11번 = Phase 2(세션스킬) 인계 입력.**

## §5. 가드레일

- **유도 금지** — 보강 질문도 열린 형태. 추측으로 구체 질문 만들어 신청자 끌고 가지 않기.
- **첨부 읽기** — 양식 URL·파일 못 열면 "직접 확인 필요" 안내(무시 X).
- **이메일 매칭** — 신청자 행은 이메일(Q3) 키(이름 중복 방지).
- **개인정보** — 세션 폴더엔 해당 신청자 정보만.
- **무상태** — 각 단계는 폴더만 보고 완결.
