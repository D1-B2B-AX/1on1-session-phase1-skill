# Phase 1 원업무 형식지 추출 인터뷰 프로토콜 v1.4

> 이 문서는 AX 프로젝트의 일환으로 AX 매니저가 신청자(같은 조직 동료)의 원업무 형식지를 추출하기 위한 1:1 세션 운영 지침서다.
> 산출물 템플릿(`Phase1_원업무_형식지_최종산출물_템플릿_v1.1.md`)을 채우는 데 필요한 정보를 끌어내는 방법을 다룬다.
> 작성 주체는 인터뷰 진행자(AX 매니저). 인터뷰 raw 데이터(녹음·세션 로그·메모 중 하나 이상)를 확보한 뒤, AI 후처리와 AX 매니저 검토를 거쳐 산출물 템플릿이 작성된다.
> 본 프로토콜은 외부 고객 인터뷰가 아니라 같은 조직 동료 인터뷰임을 전제로 한다. 따라서 신청자 물색·시간 요청·인터뷰 후 피드백 루프 초대 같은 영역은 다루지 않는다.

## 인터뷰 준비 (전제 사항)

본 프로토콜이 작동하기 위한 최소 전제. 세션 진입 전에 충족되어야 한다.

- **신청자 사전 안내 항목** (운영 영역 — 세션 약속 잡을 때 AX 매니저가 직접 안내):
  - 최근 실제 사례 1~3건 (실제 입력 자료·산출물·중간 메모)
  - 화면 공유 가능 환경
  - 기존 형식지(매뉴얼·체크리스트·SOP)가 있다면 같이 준비
- **raw 데이터 확보 의무**: 인터뷰 진행 중 녹음·세션 로그·실시간 메모 중 하나 이상을 반드시 확보한다. 형식은 자유. 보관은 AX 매니저 책임.
- **산출물 템플릿(최신본) 사본**을 인터뷰 전에 한 번 통독한 상태로 진입한다.

## 진행 방법 — 9가지 원칙

본체 9단계에 들어가기 전에, 인터뷰 진행 전반에 적용되는 원칙.

1. **들으러 가는 자리다, 설명하러 가는 자리가 아니다.** 진행자는 적게 말하고 많이 듣는다.
2. **신청자가 말한 발화는 원본 그대로 raw 데이터에 보존한다.** 후처리에서 산출물 템플릿 항목으로 매핑할 때 필요.
3. **본인(진행자)의 가설·아이디어·구현 방향은 인터뷰 중 도입하지 않는다.** 신청자가 그쪽으로 끌려가 편향된 답변을 줄 수 있다. 진짜 묻고 싶은 게 있으면 인터뷰 마지막에 짧게.
4. **인터뷰 중 발화는 차단하지 않되, 산출물 작성 시 Phase 1/2를 분리한다.** 신청자가 "이거 자동화하면 좋겠어요" 같은 발화를 해도 막지 않는다. 그 발화 자체를 raw 데이터에 기록하고, 산출물 템플릿 작성 시 Phase 1에 필요한 정보만 추출한다.
5. **가설을 검증하지 말고 사실을 수집한다.** "X 맞죠?"가 아니라 "X 어떻게 하셨어요?"로 묻는다.
6. **사용자가 제안하는 solution은 신뢰도 ↓, problem은 신뢰도 ↑.** 신청자가 "이런 기능 있으면 좋겠어요" 같은 solution 제안은 참고만 하고, 그 배경에 있는 problem을 파고든다. (영상 사례: Gmail 사용자가 "inbox+email 동시 보고 싶다" → 실제 문제는 Gmail 느림. Airbnb 게스트가 "호스트 전화번호 받고 싶다" → 실제 문제는 플랫폼 신뢰 부족.)
7. **구체적 과거 사례를 묻는다, 일반적 의견·가설적 미래는 묻지 않는다.** "보통 어떻게 하세요?"보다 "지난주 마지막 사례에서 어떻게 하셨어요?"가 더 풍부.
8. **신청자가 실제로 일하는 모습을 본다.** 가능하면 화면 공유로 시연하게 한다. 시연 중 "지금 무슨 생각 하시면서 하시나요?"로 사고 과정을 소리 내어 말하게 한다.
9. **답을 못 하면 "없는 것"으로 처리하지 않는다.** 라벨링한다(`[신청자 응답 어려움]`·`[추가 확인 필요]` 등 — 산출물 템플릿 8번·10번 참조).

## 본체 9단계

각 단계 구성:
- **목적**: 이 단계에서 끌어내려는 정보
- **A. 질문 세트** (기본 / 꼬리 / 기록 포인트 3구분)
- **단계 운영 메모** (해당 시)

---

### 1. 도입 (기존 형식지 유무 확인 포함)

**목적**: 인터뷰 분위기 형성. 신청자가 기존에 본인 업무에 대한 형식지(매뉴얼·체크리스트·SOP)를 가지고 있는지 확인. 있으면 그 자료를 raw 데이터에 추가하고 본체 진행에 출발점으로 활용.

**A. 질문 세트** (기본 2건 / 꼬리 1건)

기본 질문:
- 오늘은 [신청자]님의 [업무명] 업무가 실제로 어떻게 수행되는지 들어보러 왔어요. 편하게 떠올리시면서 답해주시면 돼요.
- 혹시 이 업무에 대해 본인이 작성하거나 팀에서 공유하는 매뉴얼·체크리스트·SOP(표준 업무 절차서)가 있나요? 있으면 같이 보면서 진행할게요. (신청자가 SOP를 모르면 "업무 순서를 정리해둔 문서 같은 거요"로 풀어 설명)

꼬리 질문:
- (기존 형식지 있음) 그 자료를 같이 볼 수 있을까요? 지금 화면 공유 가능하세요?

**기록 포인트**:
- 기존 형식지가 있다면 raw 데이터에 추가 (파일·링크·스크린샷)
- 산출물 템플릿 9번(형식지 작성 근거)에 "기존 형식지" 유형으로 등재

**단계 운영 메모**:
- 기존 형식지가 있어도 본체 단계는 모두 진행한다. 그 자료는 매핑이 다르거나 정보가 불충분할 가능성이 높으므로, 출발점으로만 활용한다.
- 라포 형성을 위해 첫 1~2분은 가벼운 안부도 OK. 단 진짜 인터뷰는 빠르게 진입.

---

### 2. 업무 정체 파악 (5요소)

**목적**: 신청자의 업무가 누가·무엇을·누구를 위해·언제·왜 하는지 파악. 산출물 템플릿 1번 채움.

**A. 질문 세트** (기본 2건 / 꼬리 2건)

기본 질문:
- [업무명] 업무는 [신청자]님이 보통 무엇을 대상으로 하시는 거예요? (5요소: 대상)
- 이 업무는 누구를 위해 / 누구한테 결과가 가는 일이에요? (5요소: 수혜자)

꼬리 질문:
- 이 업무는 얼마나 자주, 언제 하시나요? (5요소: 주기)
- 이 업무가 [신청자]님 / 팀 / 회사 입장에서 왜 중요한 일인가요? (5요소: 맥락·목적 — 동기 깊이 파기)
- 이 업무에서 다루는 대상이 전체 중 일부인가요? 그렇다면 본인이 기준을 보고 매번 고르는 건지, 아니면 원래 특정 조건의 대상만 들어오거나 처리되는 업무인지 궁금해요. 고르신다면 어떤 정보를 보고 고르세요? 정해진 조건이라면 그 조건은 어디서 정해진 건가요? (전체/일부 대상 — 관련 내용을 산출물 1·2·3·5번 중 자연스러운 곳에 기록)

**기록 포인트**:
- 산출물 템플릿 1번 5요소 (누가 / 무엇을 대상으로 / 누구를 위해 / 어떤 주기로·언제 / 어떤 맥락·목적으로)
- "누가"는 신청자 본인이 명확하면 빠른 확인만

**단계 운영 메모**:
- 신청자가 "왜 중요한가"에 "그냥 해야 해서요" 식의 답을 하면 한 번 더 캐묻는다 (모호어 가이드 C 참조). 동기는 인터뷰 전반의 톤을 결정한다.

---

### 3. 대표 사례 재현

**목적**: 최근 실제 사례 1건을 따라가며 업무 흐름의 골격을 구체적으로 끌어낸다. 추상적 설명이 아닌 구체적 행동·자료·시점.

**A. 질문 세트** (기본 3건 / 꼬리 2건 / 1건 행동 관찰)

기본 질문:
- 가장 최근에 이 업무를 하신 게 언제예요? 그 사례 1건을 같이 따라가 볼게요.
- (사례 선정 후) 그날 처음에 어디서부터 시작하셨어요? 무엇을 보고 / 무엇을 받고 시작하셨나요?
- 그 다음에 무엇을 하셨어요? (단계별로 차례차례 따라가기 — 흐름이 자연 드러날 때까지 반복)

꼬리 질문:
- 그때 무슨 생각 하시면서 그 단계를 하셨어요?
- 혹시 그 단계 화면이나 자료를 지금 같이 볼 수 있을까요?

행동 관찰 (화면 공유 가능 시):
- 지금 화면 보면서 그때 하셨던 방식 그대로 한번 보여주실 수 있어요? 하시면서 "여기선 이걸 본다", "이때는 이렇게 한다" 같은 판단·행동이 있으면 짚어주세요.

**기록 포인트**:
- 산출물 템플릿 4번 본문 표 (순서 / 단계명 / 실제로 한 일 / 이때 본 자료·화면 / 단계에서 확인한 것 / 단계 결과 / 판단·선택 여부)
- 사례에만 특이한 부분(평소와 다름)은 4번 특이사항 메모에 분리
- 시연·화면 공유 영상은 raw 데이터에 보관

**단계 운영 메모**:
- 이 단계가 인터뷰의 가장 중요한 출구. 사례 재현이 풍부할수록 4~7단계가 자동 잘 풀린다.
- "보통 이렇게 해요"라는 일반화 답변이 나오면 "지난번 마지막 사례에서는 어떻게 하셨어요?"로 구체화한다 (영상 영감: specific past events > general opinions).

---

### 4. 입력·흐름 구조화

**목적**: 3단계에서 드러난 입력 자료와 흐름을 산출물 템플릿 표 형식으로 정리. 비공식 채널(구두·메신저·전화·회의 발언) 누락 방지. 사례에 안 나온 흐름·자료 추가 발굴.

**A. 질문 세트** (기본 3건 / 꼬리 3건)

기본 질문:
- 지금까지 말씀하신 흐름을 제가 단계별로 정리해볼게요. 보통 이 일을 시작하시는 신호·계기는 뭐예요? (시작 조건 — 산출물 2번)
- "이 정도면 끝났다"라고 인식하시는 시점은 언제예요? (완료 조건 — 산출물 2번)
- 이 업무에 사용하시는 자료·정보는 [3단계에서 언급한 것 외에] 또 뭐가 있어요? (특히 구두 보고·메신저·전화·회의 발언도 포함)

꼬리 질문:
- 그 자료에서 실제로 보시는 부분은 전체예요, 일부예요? 어디를 주로 보세요?
- 그 자료가 자주 비거나, 빠지거나, 형식이 달라지는 경우가 있어요?
- 그 자료를 받자마자 바로 쓰세요? 아니면 받기 전에 뭔가 한번 확인하세요?
- 업무하시면서 본인이 직접 점수·등급·가능성·상태 같은 걸 매겨서 시스템이나 시트에 입력하시는 게 있나요? (신청자 직접 입력 판단값 — 산출물 5번 판단으로 기록)

**기록 포인트**:
- 산출물 템플릿 2번 (시작·완료 조건 + 관련 자료 + 비고)
- 산출물 템플릿 3번 표 (입력 자료 / 형태·채널 / 출처 / 실제로 보는 부분 / 필수 여부 / 자주 빠지거나·변동되거나·달라지는 부분 / 사용 전 확인 사항)

**단계 운영 메모**:
- "실제로 보는 부분"이 핵심 — 자료 전체를 다 보는 게 아니라 일부 필드·구간만 보는 경우 명문화.
- 비공식 채널은 신청자가 자주 잊는다. "혹시 슬랙이나 이메일로 받으시는 것도 있나요?"·"회의에서 말로 전달받는 건요?"·"세일즈맵 노트에 적어두시는 것도 있나요?" 식으로 명시적으로 물어본다.
- **단계 종료 시 부분 요약 확인 루프**: 진행자가 들은 흐름을 짧게 되돌려 신청자에게 확인. 예: "제가 이해한 흐름은 A → B → C이고, B 단계에서 X 자료의 Y 값을 주로 보시는 걸로 이해했어요. 맞나요?" 오해·누락이 단계 내에서 즉시 정정됨. (9단계 마무리 확인은 전체 요약이고, 본 루프는 단계별 부분 요약 — 둘 다 진행.)

---

### 5. 판단 지점·기준 추출

**목적**: 흐름 중 선택·보류·확인·수정이 발생한 지점과 그 기준을 추출. 산출물 5번 카드 채움. 핵심 단계 — 가장 깊은 인터뷰 비중.

**A. 질문 세트** (기본 2건 / 꼬리 5건)

기본 질문:
- 이 흐름 중에서, "그냥 진행할까 / 멈출까 / 누군가에게 확인할까" 같이 결정해야 하는 순간이 있었나요? 떠오르는 순서대로 다 말씀해주세요.
- 그 순간에 어떻게 결정하셨어요? 그 결과를 선택한 이유가 뭐예요?

꼬리 질문 (각 판단 지점마다 반복):
- 그때 무엇을 보고 결정하셨어요? (참고 정보)
- 반대로 처리했던 사례가 있었나요? 그 경우는 뭐가 달랐어요? (다르게 처리되는 경우)
- 이 조건이 조금 달라지면 결과도 달라지나요? (경계값)
- 다른 사람이 같은 상황을 보면 어떻게 판단해야 하나요? (기준 일반화)
- (모호어 등장 시) "적절히"라고 말씀하셨는데, 구체적으로 어떤 상태면 적절한 거예요? (모호어 가이드 C 참조)

**기록 포인트**:
- 산출물 템플릿 5번 카드 (판단 지점 N: 발생 상황 / 실제로 고민한 질문 / 선택한 결과 / 참고한 정보 / 이렇게 판단한 기준 / 다르게 처리되는 경우)
- 4번 사례에서 발견한 판단 + 사례 밖에서 발견한 판단(일반 패턴·다른 사례) 모두 포함
- 기준이 풀리지 않으면 → 산출물 10번 미확인·보완 필요 항목으로 이동 (유형 `기준 표현 미완성` 또는 `암묵지·직감 영역`)

**단계 운영 메모**:
- 신청자가 "그냥 직감으로 해요"·"케바케예요"라고 답하면 → 막힘 대응표 B 참조 (비교 사례·diff·경계값 질문 활용).
- 포함/제외/보류, 우선순위, 수정 필요 여부, 추가 확인 여부, 전달 대상 결정 등이 모두 판단에 해당.
- 판단이 10건을 넘어가면 산출물 템플릿에서 표로 압축 (작성 단계 영역).
- **판단 기준 조건문 변환 확인**: 신청자 답변을 진행자가 조건문으로 바꿔 되돌려 확인한다. 예: "제가 기준을 조건문으로 바꿔보면, 'X이면 진행, Y이면 보류, Z이면 담당자에게 확인'으로 이해해도 될까요?" 이 단계에서 조건문화가 안 되면 산출물 10번 미확인·보완(`기준 표현 미완성` 유형)으로 이동.
- **단계 종료 시 부분 요약 확인 루프**: 진행자가 들은 판단 지점·기준을 짧게 되돌려 신청자에게 확인. 예: "이 흐름에서 핵심 판단은 [1~2개]이고, 각각 [조건]에 따라 [결과]로 가는 걸로 이해했어요. 맞나요?" (9단계 마무리 확인은 전체 요약이고, 본 루프는 단계별 부분 요약 — 둘 다 진행.)

### 6. 산출물·품질 조건 확인

**목적**: 이 업무의 출구(결과물)와 좋은 결과물의 조건을 명확화. 산출물 6번 + 7번 채움.

**A. 질문 세트** (기본 3건 / 꼬리 3건)

기본 질문:
- 이 업무가 끝나면 무엇이 만들어지나요? 그게 어떤 형태예요? (문서? 표? 메신저 메시지? 시스템 기록? 노트? 구두 공유?)
- 그 결과물은 어디에 저장되고, 누구한테, 어떤 방식으로 전달돼요?
- "이건 잘 만들었다"고 느끼시는 기준은 뭐예요?

꼬리 질문:
- 정해진 양식·템플릿이 있나요? 있다면 어디 있어요?
- 그 결과물의 통과 기준이 명확하게 있어요? (예: 오타 0개여야 통과인지, 의미 전달되면 acceptable인지)
- 그 기준은 누가, 언제, 어떻게 확인해요? 안 맞으면 어떻게 처리해요?

**기록 포인트**:
- 산출물 템플릿 6번 (산출되는 것 / 형태 / 템플릿·양식·규칙 / 필수 구성요소 / 저장 위치 / 전달 대상 / 전달 방식 / 완료로 인정되는 상태)
- 산출물 템플릿 7번 카드 (조건 N: 충족 조건 / 통과 기준·수준 / 기준 출처 / 확인 방식 / 미충족 시 처리)

**단계 운영 메모**:
- "충족해야 하는 조건"(무엇)과 "통과 기준·수준"(어느 정도면 통과)을 의식적으로 분리해 묻는다.
- 기준이 명시 매뉴얼·관행·수신자 피드백·본인 판단 중 어느 것인지 출처도 같이 받는다.

### 7. 주의·실수·예외·애매한 상황 확인

**목적**: 정상 흐름 밖의 구간 발굴. 산출물 8번 4섹션 채움. 핵심 단계 — 신청자가 자연 인지하지 못한 영역을 끌어낸다.

**A. 질문 세트** (기본 2건 / 꼬리 6건)

기본 질문:
- 이 업무를 하실 때 "이건 진짜 조심해야 한다"·"여기를 잘 봐야 한다"고 느끼시는 부분이 있어요?
- 정상 흐름대로 안 풀리는 상황이 있어요? (예: 입력이 빠지거나, 기준이 충돌하거나, 담당자가 없거나, 자료 형식이 달라지거나, 수신자가 반려하거나)

꼬리 질문 (네 영역 각각 1~2개씩):
- (주의할 점) 이걸 놓치면 큰 문제가 생기는 부분이 있어요? 왜 큰 문제예요?
- (자주 생기는 실수) 본인이나 다른 사람이 자주 하는 실수가 있어요?
- (예외) 평소와 다르게 처리한 가장 최근 사례가 떠오르세요?
- (애매) "이건 어떻게 해야 할지 모르겠다"고 느끼신 적이 있어요?
- 그런 상황이 생기면 어떻게 처리하셨어요?
- 그 처리가 정해진 절차예요, 그때그때 판단이에요?
- (권한·정책) 업무 중에 권한이나 회사 정책 때문에 원래 하려던 방식이 막혀서 다른 방법으로 돌아가신 적 있나요?
- (권한·정책) 자료를 가져오거나 결과물을 전달·저장할 때 권한·보안·개인정보 때문에 제약받는 부분이 있나요?

**기록 포인트**:
- 산출물 템플릿 8번 4섹션 (주의할 점 / 자주 생기는 실수 / 예외 상황 / 애매한 상황) — 각 섹션 사례별로 (상황 / 왜 중요 / 처리)
- 빈 섹션은 라벨 사용: `[확인된 사례 없음]` / `[신청자 응답 어려움]` / `[추가 확인 필요]`
- 권한·정책 제약은 발생 지점(산출물 3·4·6번)에 `[권한·정책]` 태그로 기록하고 11번에 종합한다. 실제로 부딪힌 제약만 기록(추정 X).

**단계 운영 메모**:
- 신청자가 "다 설명했어요"·"별로 없어요"라고 빠르게 끝내면 → 막힘 대응표 B 참조. 사례 부재가 아니라 인지 부재일 가능성.
- 예외는 "예외를 떠올려보세요"가 아니라 "정상 흐름대로 안 됐던 가장 최근 사례"로 묻는다 (specific past events > general).

### 7.5 페인·개선 희망 (인터뷰 후반)

**목적**: 이 업무에서 느끼는 병목·불편을 *문제(problem)* 형태로 수집. 산출물 11번(페인·AX 의도) 신호이며 Phase 2가 참고한다. 원업무 복원(1~7단계)이 끝난 뒤 후반에 둔다.

**A. 질문 세트** (기본 2건 / 꼬리 1건)

기본 질문:
- 이 업무 하시면서 제일 시간을 잡아먹거나 번거로운 구간은 어디예요?
- 최근에 이거 하다가 "아, 이건 좀…" 했던 순간 있었어요? (일반론보다 구체 사례로)

꼬리 질문:
- (바꾸고 싶다는 답이 나오면) 그게 지금 왜 불편하세요? 지금은 어떻게 하세요? (솔루션 발화 → 배경 problem으로 되돌리기)

**기록 포인트**:
- 산출물 11번(페인·AX 의도)에 요약. 관련되면 8번(애매한 상황)과 연결.

**단계 운영 메모**:
- 반드시 인터뷰 *후반*(원업무 복원 1~7단계 후)에. 중간에 물으면 신청자가 솔루션 모드로 빠져 원업무 그림이 번거로운 것 위주로 쏠린다.
- 9원칙 6번대로: *문제(problem)*는 신뢰도↑, *해결책(solution)* 제안은 참고만 — solution 발화는 받아 적고 그 배경 problem을 캐묻는다.

### 8. 미확인·보완 정리

**목적**: 이번 세션에서 못 채운 항목·풀지 못한 부분을 라벨링. 산출물 10번 채움. 후속 확인·자료 확보 트리거.

**A. 질문 세트** (기본 1건 / 꼬리 2건)

기본 질문:
- (인터뷰 진행 중 라벨링한 항목들을 같이 정리하면서) 지금까지 들은 내용 중에 "이건 자료 한번 더 보면 알 수 있어요"·"팀원한테 한번 더 물어볼게요"·"제가 시간 두고 정리해볼게요" 같은 게 있었어요?

꼬리 질문:
- 그건 어떻게 해야 풀릴 것 같아요? (자료 확인 / 후속 질의 / 본인 정리 / 등)
- 그게 풀리는 데 얼마나 걸릴 것 같아요?

**기록 포인트**:
- 산출물 템플릿 10번 표 (항목 / 유형 [자료 추가 확인 / 기준 표현 미완성 / 신청자 확인 필요 / 근거 부족 / 암묵지·직감 영역] / 왜 남았는가 / 보완 방법 / 상태)
- 인터뷰 중 D(임시 보류 메모 양식)에 누적한 항목들이 모두 여기로 옮겨진다.

**단계 운영 메모**:
- 신청자에게 "보완해주세요"라는 부담을 주지 않는다. 후속 확인은 AX 매니저가 별도 채널(DM·자료 추가 확인)로 가져갈 수 있다는 톤으로.
- 빈 표는 `[현재까지 보완 필요 항목 없음]` 라벨로 표시 (산출물 v1.1 안내 참조).

### 9. 마무리 확인

**목적**: AX 매니저가 이해한 원업무 구조를 신청자에게 짧게 되돌려 확인. 오해·누락 즉시 발견.

**A. 질문 세트** (기본 1건 / 꼬리 1건)

기본 질문:
- 제가 들은 걸 짧게 정리해보면, [신청자]님은 보통 [5요소 한 줄] 일을 하시는데, 흐름은 [3~4단계 핵심], 가장 중요한 판단은 [1~2개], 자주 신경 쓰시는 부분은 [1~2개]예요. 맞게 이해한 거예요?

꼬리 질문:
- 혹시 빠진 부분이나, 제가 잘못 이해한 부분 있어요?

**기록 포인트**:
- 잘못 이해한 부분 → 즉시 수정 (raw 데이터에 정정 기록)
- 빠진 부분 → 산출물 10번 미확인·보완 또는 즉석 추가 질의

**단계 운영 메모**:
- 형식지 전체를 신청자에게 보여주는 게 아니다. 핵심만 1~2분 안에 압축해 되돌린다.
- 빠진 부분이 많으면 후속 확인 약속 (별도 채널 DM 등).

## 보조 도구 B. 막힘 패턴 대응표 (v1 — 가설)

> **이 표는 v1 가설이다.** 실제 세션 사용 사례가 누적되지 않은 상태로 작성되었다. 실제 세션 후 AI와 함께 검증·보강해 v2로 갱신한다. 현재는 진행자가 막혔을 때 참고할 후보 도구 목록.

| 막힘 패턴 (신청자 발화) | 가능한 전환 도구 (v1 가설) | 도구 예시 질문 |
|---|---|---|
| "그냥 보면 알아요" / "직감이에요" | 비교 사례 / diff 질문 | "최근 비슷한 사례 2~3건 떠올려보세요. 어떤 차이 있었어요?" / "왜 A는 통과시키고 B는 보류하셨어요?" |
| "케바케예요" / "그때그때 달라요" | 경계값 질문 / 최근 3건 비교 | "어떤 조건이면 A로 가고, 어떤 조건이면 B로 가는지 떠오르는 경계가 있어요?" / "최근 3건 떠올려보시면 어떻게 갈렸어요?" |
| "팀장님 판단이에요" / "PM이 결정해요" | 1차 필터링 확인 / 실제 처리 기준 확인 | "팀장님께 올리기 전에 [신청자]님이 한번 보고 가시는 거 있어요?" / "어떤 상태면 팀장님께 가고, 어떤 상태면 본인이 처리하세요?" |
| "흐름이 잘 떠오르지 않아요" | 화면 공유 / 사례 재현 | "지금 화면 보면서 최근 사례 1건 그대로 따라가 볼게요" / "지난주에 하셨던 거 화면 같이 봐도 될까요?" |
| "기준은 있는데 말로 못 해요" | decision tree 역추적 / Yes-No 분기 따라가기 | "처음 이 자료 보시면 무엇을 먼저 확인하세요? 그게 X면 다음에 뭘 하세요? 아니면요?" |
| "다 설명한 것 같아요" / "별로 없어요" (예외·실수 단계) | 사례 강제 / 시점 좁히기 | "최근 한 달 안에 평소와 달랐던 사례 떠오르세요?" / "마지막으로 다시 작업한 적이 언제예요?" |
| "그게 자동화되면 좋겠어요" / "이거 LLM 쓰면 빠르겠죠" | 원업무로 되돌리기 (제약 X 톤) | "그 부분 메모해둘게요. 지금은 어떻게 일하시는지만 한번 더 봐도 될까요?" |
| "왜 그렇게 하는지 모르겠어요" / "그냥 하던 대로요" | 동기 파기 / 시작 시점 추적 | "처음에 누가 그렇게 하라고 알려주셨어요?" / "이걸 안 하면 어떤 문제가 생겨요?" |
| 침묵 (말 안 하고 생각만) | 침묵 견디기 / 5초 룰 | (진행자가 먼저 말 X — 5초 기다리기. 신청자가 생각 끝내고 자연 답변 시작하게.) |
| 옆길 (다른 업무·잡담) | 부드럽게 본 업무로 복귀 | "그것도 흥미로운데, 일단 [업무명] 쪽 흐름부터 마저 정리할게요. 아까 [N단계]까지 말씀하셨던 거 같은데..." |

### 도구 해설

위 표에 등장하는 핵심 도구는 이름만 보면 의미 파악이 어려울 수 있어 별도 설명한다.

- **diff 질문**: 정답을 찾는 도구가 아니다. **비슷한 두 사례가 왜 다르게 처리됐는지 비교**해 신청자도 의식하지 못한 판단 기준을 드러내는 도구. 예: "지난주 사례 A와 이번 주 사례 B가 비슷해 보이는데, A는 통과시키고 B는 보류하셨네요. 두 사례에서 어떤 부분이 달랐어요?"

- **decision tree 역추적**: 미리 만들어진 의사결정 트리를 적용하는 게 아니다. **신청자가 실제로 확인하는 순서를 Yes/No 또는 조건 분기로 거꾸로 복원**하는 방식. 예: "처음 이 자료 받으시면 무엇을 먼저 확인하세요? 그게 X면 다음에 뭘 하세요? 아니면 어떻게 하세요?" 식으로 진행자가 트리를 따라가며 그리는 것.

- **침묵 견디기**: 신청자가 생각 중일 때 진행자가 먼저 말 채우지 않는 것. 5초 정도 기다리면 신청자가 더 깊은 답을 자연 시작하는 경우가 많다. 진행자의 본능적 채움이 인터뷰 깊이를 얕게 만드는 함정.

- **경계값 질문**: 판단 결과가 갈리는 조건의 *경계*를 찾는 질문. 신청자는 양극단(완전 통과/완전 보류) 사례는 잘 떠올리지만 경계 근처 사례는 모호하게 답한다. 의도적으로 경계를 물어 기준을 정량화한다. 예: "수치가 어디까지면 통과시키시고, 어디부터면 보류하세요?"

## 보조 도구 C. 모호어/회피어 재질문 가이드

신청자 발화에 다음 어휘가 등장하면 진행자가 멈추고 재질문한다. 모호어를 조건·구체 사례로 풀어내는 게 형식지 추출의 본질.

### 모호어 재질문 공통 패턴 (4단계)

개별 어휘별 질문을 외우기보다 다음 4단계를 순서대로 적용하면 대부분의 모호어를 풀 수 있다.

1. **그 말이 뜻하는 상태를 묻는다.** "그게 어떤 상태를 뜻하시는 거예요?"
2. **최근 실제 사례를 묻는다.** "최근에 그렇게 처리하신 사례가 있어요?"
3. **반대 사례나 경계값을 묻는다.** "반대로 처리하신 적이 있어요?" / "어디부터 다르게 처리되세요?"
4. **다른 사람이 따라 하려면 어떤 조건을 봐야 하는지 묻는다.** "이걸 처음 하는 사람이 똑같이 하려면 무엇을 봐야 할까요?"

아래 어휘별 예시는 이 4단계의 구체적 적용 사례다.

### 모호어·회피어 사전 + 재질문 예시

| 등장 어휘 | 의미 추정 | 재질문 예시 |
|---|---|---|
| "적절히" / "적당히" | 조건이 있지만 말로 못 함 | "어떤 상태면 적절한 거예요? 적절하지 않으면 어떻게 보여요?" |
| "중요하면" / "중요한 건" | 우선순위 기준 있음 | "무엇을 보고 중요한지 판단하세요? 중요하지 않은 건 어떤 거예요?" |
| "자연스럽게" / "그냥" | 무의식적 패턴 | "그 단계 직전에 뭘 보고 그렇게 하시는 거예요? 처음에는 어떻게 알게 되셨어요?" |
| "상황에 따라" / "케바케" | 분기 조건 있음 | "어떤 상황이면 A, 어떤 상황이면 B인지 떠오르는 경계 있어요?" |
| "보통은" / "원칙적으로는" | 예외가 본질일 수 있음 | "원칙은 그렇고, 실제로 다르게 한 적이 있어요? 그게 언제였어요?" |
| "원래 그렇게 해요" | 학습 시점 / 이유 파악 필요 | "처음에 누가 그렇게 알려주셨어요? 왜 그렇게 하는지 들으신 적 있어요?" |
| "그냥 보면 알아요" / "직감" | 암묵지 영역 | (막힘 대응표 B "직감" 항목 참조) |
| "팀장님 / PM 판단" | 1차 필터링 가려져 있을 수 있음 | (막힘 대응표 B "팀장님 판단" 항목 참조) |
| "어떻게든" / "여러 가지 방법" | 구체 사례 부재 | "최근에 어떻게든 처리하신 사례 하나 떠올려주실 수 있어요?" |
| "간단해요" / "별거 아니에요" | 암묵지 가장 많이 숨은 영역 | "간단해 보이는데, 처음 하시는 분이 똑같이 하려면 뭘 알아야 할까요?" |
| "효율적으로" / "잘" | 평가 기준 있음 | "효율적인 결과랑 그렇지 않은 결과를 어떻게 구분하세요?" |

### Follow-up 질문 3종 (Gustav 영상 영감)

모호어 외에도, 답이 추상적이거나 짧을 때 사용:

- **"무슨 뜻이세요?"** (What do you mean by that?) — 의미 명확화
- **"좀 더 자세히 말씀해주실래요?"** (Tell me more about that) — 깊이 파기
- **"그게 [신청자]님께 왜 중요해요?"** (Why is that important to you?) — 동기 파기

## 보조 도구 D. 세션 중 임시 보류 메모 양식

인터뷰 진행 중 발견된 보류 항목을 즉시 기록한다. 다만 인터뷰 중에 표를 채우는 건 부담이 크므로 **실시간 퀵 메모**와 **후처리 정리 표** 2단으로 분리한다.

### D-1. 실시간 퀵 메모 (인터뷰 중)

인터뷰 흐름을 끊지 않도록 한 줄 짧은 자유 형식으로 기록. 진행자 메모장이나 노션에 한 줄씩 적기. 정해진 라벨 코드는 없다 — 진행자가 알아볼 수 있는 형태면 충분.

**자주 등장하는 메모 유형** (이 유형으로만 한정되지 않음 — 인터뷰 중 새 유형 발견 시 자유 추가):
- 신청자 응답 어려움 (직감·케바케 등)
- 예외·추가 사례 확인 필요
- 입력 자료·자료 위치 확인 필요
- 후속 자료·DM 확인 필요

각 메모는 `[N단계 항목] 한 줄 메모 — 후속 처리 방향` 형식을 권장. 단계 번호만 표시하면 후처리에서 빠르게 분류 가능.

### D-2. 후처리 정리 표 (8단계 또는 세션 직후)

D-1의 퀵 메모를 8단계 미확인·보완 정리 단계 또는 세션 종료 직후 표 형식으로 정리. 산출물 10번 표로 옮길 때 그대로 사용 가능한 형태.

```markdown
| 시점(단계·시각) | 항목 | 라벨 | 메모 |
|---|---|---|---|
| 예: 5단계, 14:25 | 판단 지점 "긴급 우선순위 기준" | [신청자 응답 어려움] | "직감으로 한다" 답변. 후속 질문 필요 |
| 예: 7단계, 14:48 | 예외 상황 "수신자 반려 시" | [추가 확인 필요] | 사례 1건만 들었음, 추가 사례 더 있을 가능성 |
```

**작성 힌트**:
- 라벨은 산출물 v1.1과 동일 3종: `[확인된 사례 없음]` / `[신청자 응답 어려움]` / `[추가 확인 필요]`
- D-1 메모의 "후속 처리 방향" 문구가 D-2의 메모 칸 + 라벨로 변환된다.
- 산출물 10번 표로 옮길 때 유형 매핑 참고: 직감·기준 풀이 못함 → `암묵지·직감 영역` 또는 `기준 표현 미완성` / 자료·사례 확인 → `자료 추가 확인` / 신청자에게 다시 물어볼 부분 → `신청자 확인 필요`

## 진행자 주의 사항 (인터뷰 함정 회피)

### 진행자가 하지 말아야 할 것

- **본인 가설·아이디어·구현 방향을 인터뷰 중 도입하지 않는다.** 신청자가 동조하게 됨. 진짜 묻고 싶은 게 있으면 마지막에.
- **Yes/No 질문에 머물지 않는다.** "이거 자주 하세요?" (X) → "최근 한 달에 몇 번 정도 하셨어요?" (O)
- **두 질문을 한 번에 묻지 않는다.** "그때 어떻게 결정하시고, 누구한테 보고하셨어요?" (X) → 한 번에 하나씩.
- **신청자가 제안한 solution을 그대로 받아들이지 않는다.** "이 기능 있으면 좋겠어요" 발화는 raw 데이터에 기록만 하고, 그 배경의 problem을 묻는다 (영상 사례: Gmail "두 화면" 요청 → 실제는 Gmail 느림).
- **"~하시는 거 맞죠?" 식 가설 검증 질문을 하지 않는다.** "어떻게 하세요?"로 묻고 신청자가 표현하게 한다.
- **"이런 기능이 있으면 어떻게 쓰실 것 같으세요?" 묻지 않는다.** 가설적 미래 답변은 신뢰도 낮다. 구체적 과거 사례를 묻는다.
- **신청자가 "다 설명했어요"라고 해도 그대로 끝내지 않는다.** 8단계 라벨링·9단계 마무리 확인을 통해 한 번 더 짚는다.

### 진행자가 해야 할 것

- **들으러 왔다고 의식적으로 인지한다.** 진행자가 말한 시간 > 신청자가 말한 시간이면 인터뷰 실패.
- **침묵을 견딘다.** 신청자가 생각 중일 때 진행자가 먼저 말 채우지 않는다. 5초 정도 기다리기.
- **구체적 과거 사례를 끌어낸다.** "보통 어떻게 하세요?"보다 "지난주 마지막 사례에서는?"이 더 풍부.
- **신청자가 실제 행동하는 모습을 본다.** 화면 공유로 시연 + 사고 과정 소리 내어 말하기.
- **신청자가 솔루션 제안하면 problem으로 돌린다.** "그 부분 메모할게요. 그 전에 지금은 어떻게 일하시는지 한 번 더 볼까요?"
- **모호어가 등장하면 멈추고 재질문한다.** C 가이드 참조.
- **막혔다고 느끼면 즉시 도구 전환한다.** B 막힘 대응표 참조. 같은 질문 반복은 신청자 부담만 키움.

```
[역할]
당신은 AX 매니저를 도와 1:1 세션 raw 데이터를 Phase 1 원업무 형식지로 정리하는 작업을 합니다. 목표는 구현 방향을 정하기 전에, 신청자가 현재 원업무를 실제로 수행하는 방식을 충분한 깊이로 복원하는 것입니다. 추출 깊이의 기준은 "아무것도 모르는 후임자가 이 문서만 보고도 해당 업무를 수행할 수 있는 수준"입니다. 이 업무를 AX화하기 위한 구현 방식이나 자동화 방향은 여기서 단정짓지 않습니다(그건 다음 단계인 Phase 2의 일입니다).

[입력]
- 세션 raw 데이터: 첨부한 녹음 전사 / 세션 로그 / 진행자 메모
- 산출물 템플릿: 첨부한 빈 양식(11개 섹션)

[작업]
첨부한 raw 데이터를 읽고, 첨부한 템플릿의 11개 섹션을 채웁니다. 각 섹션에 적힌 "작성 힌트"를 따릅니다.
섹션: 1.업무 정의(5요소) / 2.시작·완료 조건 / 3.입력 자료 / 4.대표 사례 기반 업무 흐름 / 5.판단 지점·기준 / 6.산출물·완료 기준 / 7.좋은 결과물 조건 / 8.주의·실수·예외·애매한 상황 / 9.형식지 작성 근거 / 10.미확인·보완 필요 항목 / 11.보충 메모

[반드시 지킬 규칙]
1. raw에 있는 내용을 기반으로 작성합니다. raw에서 합리적으로 추론할 수 있는 부분은 추론해도 되지만, 추론한 내용은 끝에 "(추론)"으로 표시합니다. 추론조차 어려운 항목은 비우고 10번(미확인·보완)에 기록합니다. raw와 무관한 내용을 임의로 지어내지 않습니다.
2. 신청자 발화는 가능한 한 원본 표현을 보존합니다(읽기 좋게 다듬되 의미를 바꾸지 않습니다).
3. 못 채운 항목은 10번에 유형을 분류해 적습니다: 자료 추가 확인 / 기준 표현 미완성 / 신청자 확인 필요 / 근거 부족 / 암묵지·직감 영역. (`암묵지·직감 영역`은 끝까지 풀리지 않은 잔여에만 씁니다.)
4. 주의어 검사: "LLM / API / MCP / 자동화 / 효율화 / 최적화" 같은 표현이 나오면, 그게 (a) 원업무 사실·환경인지(예: "구글 공유 드라이브에 저장한다"는 사실), 아니면 (b) 이 업무를 AX화하기 위한 구현 방법·방향 단정인지(예: "이건 MCP로 연결하면 된다") 구분합니다. (a) 원업무 사실·환경은 그대로 형식지에 보존합니다. (b) 구현 방법·방향 단정은 형식지 본문에 적지 않습니다. 단 raw에 신청자나 AX 매니저가 직접 남긴 구현 관련 발화·인사이트는 버리지 말고 11번(보충 메모)에 "Phase 2 참고 메모(비확정)"로 분리합니다. 당신(AI)이 새로 판단한 구현 방식은 적지 않습니다.
5. 권한·정책·보안·개인정보 제약은 발생 지점(3·4·6번)에 [권한·정책] 태그를 붙이고 11번에 종합합니다. 실제로 부딪힌 제약만 적고, "구현하면 막힐 것" 같은 추정은 적지 않습니다.
6. 신청자가 직접 매겨 시스템·시트에 입력하는 판단값(점수·등급·가능성·상태 등)은 5번(판단 지점·기준)에 기록합니다.
7. 전체 대상 중 일부만 다루는 업무면, 전체 규모와 일부만 처리하는 기준을 1·2·3·5번 중 자연스러운 곳에 적습니다(위치 강제 없음).
8. 8번의 네 영역(주의할 점 / 자주 생기는 실수 / 예외 상황 / 애매한 상황) 중 raw에서 사례를 못 찾은 영역은 비우지 말고 라벨을 답니다: [확인된 사례 없음] / [신청자 응답 어려움] / [추가 확인 필요].
9. 모호한 표현("적절히 / 중요하면 / 자연스럽게" 등)이 조건으로 풀리지 않은 채 남아 있으면 그대로 옮기지 말고 10번(기준 표현 미완성)에 표시합니다.

[출력]
1. 채워진 산출물 템플릿 전체(11개 섹션, Markdown). 파일명은 `Phase1_형식지_{신청자}_{업무명}.md`로 합니다(헤더 메타데이터의 신청자·업무명 사용). 파일을 직접 쓸 수 있는 환경이면 **첨부한 raw 데이터와 같은 폴더**에 이 파일명으로 저장합니다 — 케이스별 raw·형식지·후속(Phase 2) 산출물이 한 폴더에 모이고, Phase 2 단계가 이 파일을 그대로 찾아 읽을 수 있도록 하기 위함입니다. raw가 여러 개면 그중 하나가 있는 폴더에 저장합니다. 파일을 쓸 수 없는 환경(웹·일부 도구)이면 본문을 그대로 출력하고, 같은 파일명으로 raw와 같은 폴더에 저장하도록 안내합니다.
2. 그 아래에 "★ 작성자 검토 필요 목록": 추론으로 채웠지만 불확실하거나 raw가 모호해 AX 매니저가 직접 확인해야 할 지점을 (섹션 번호 · 무엇이 불확실한지 · 왜) 형태로 나열합니다.
```

→ 위 프롬프트는 skill이 아니라 그대로 복사해 쓰는 운영 프롬프트다. 후처리가 충분히 반복·안정되면 그때 skill화를 재검토한다.
