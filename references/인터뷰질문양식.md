# 인터뷰질문 양식 (= 세션 운영 콘솔)

> ※ 이 파일은 [전] 단계가 생성하는 `인터뷰질문_{신청자}_{업무}.md`의 **양식 템플릿**이다. 세션별로 이 형식(코드블록 질문 흐름·`├└` 분기·`·` 순차·`→` 다음·`- **다음**` 넘어가기)에 맞춰 생성한다. 구글폼이 채운 칸은 *확인만*, 깊이 칸은 ★로 강조.

> **세션 중 보는 진행 화면이다.** 위→아래로 따라가며 진행.
> **질문 흐름은 코드블록 안**, **넘어가기 기준은 코드블록 밖**.
> 코드블록 기호 — `[라벨]`: 진행 상황 / `├ └`: 답변 분기(정상 진행 + 막히면 프로토콜 특정 항목으로) / `·`: 이어서 묻는 질문·보조 / `→`: 다음 행동
> 코드블록 밖 — `- **다음**`: 다음 단계로 넘어가기 기준
> 배경·해설은 `인터뷰 프로토콜`(교과서). 막힘·모호어 대응은 프로토콜 **B(막힘 대응표)·C(모호어 가이드)** 참조.
> 사용 주체는 AX 매니저. 단계 번호·태그·산출물 번호는 진행자 내부용 — 신청자 앞에서 입에 담지 않는다.
> 막힌 건 무리해서 메모하려 애쓰지 말고 진행에 집중한다. 미확인·보완 항목은 세션 후 raw 리뷰(AI 후처리 포함)에서 드러나고 정리된다.

**세션 전**: □ raw 기록 준비(녹음/메모)  □ 콘솔·프로토콜 B/C 화면 준비  □ 사전 안내 완료(사례·자료·화면 공유·기존 형식지)

---

### 1. 도입 + 기존 형식지 유무   기록: 산출물 9번

```text
[열기] "오늘은 [업무명] 업무가 실제로 어떻게 수행되는지 들어보러 왔어요. 편하게 떠올리며 답해주시면 돼요."

[기존 형식지] "이 업무에 대해 작성하셨거나 팀에서 공유하는 매뉴얼·체크리스트·SOP(표준 업무 절차서) 같은 게 있나요?"
   ├ 있음 → "그 자료 같이 볼 수 있을까요?"   (raw에 추가, 본체는 그대로 진행)
   └ 없음 → 2단계로
   · SOP를 모르면: "업무 순서를 정리해둔 문서 같은 거요"
```

- **다음** — 업무 큰 그림이 잡히면 2단계

---

### 2. 업무 정의 (5요소)   기록: 산출물 1번

```text
[핵심 질문]
   · "이 업무는 보통 무엇을 / 누구를 다루는 일이에요? (자료·고객·거래 건 등)"
   · "누구를 위해 / 누구한테 결과가 가는 일이에요?"

[이어서] "얼마나 자주, 언제 하시나요?"
         "이게 [신청자]님·팀·회사 입장에서 왜 중요한 일인가요?"
   └ "그냥 해야 해서요" 식 답 → 프로토콜 C (왜 중요한가 캐묻기)

[전체 중 일부만 다루나] "다루는 대상이 전체 중 일부인가요?"
   ├ 본인이 매번 고름 → "어떤 정보를 보고 고르세요?"
   └ 원래 그 조건만 들어오거나 처리함 → "그 조건은 어디서 정해진 건가요? (사업부 원칙? 업무 정의?)"
```

- **다음** — 5요소 윤곽 잡히면 3단계 (빈 요소는 미확인 표시 후 진행)

---

### 3. 대표 사례 재현 (★핵심)   기록: 산출물 4·3·5번

```text
[시작] "최근에 이 업무 하신 사례 하나로 따라가 볼게요. 그날 처음에 뭘 보고 / 뭘 받고 시작하셨어요?"
   ├ 구체적으로 답하면 → "그 다음엔 무엇을 하셨어요?"   (흐름 끝까지 반복)
   ├ 일반론 "보통은…" → "지난번 마지막 사례에선 어떻게요?"   (사례로 되돌림)
   └ 흐름이 안 떠오르면 → 화면 공유로 재현

[화면 있으면] "지금 화면 보며 그때처럼 해보실래요? '여기선 이걸 본다' 같은 판단·행동 짚어주세요."

[대표 사례 확인] "이 사례가 평소 이 업무랑 비슷한 편이에요, 좀 특수한 경우였어요?"
   └ 특수하면 → 평소 사례도 하나 더 떠올려 비교
```

- **다음** — 흐름이 5~12단계로 잡히면 4단계

---

### 4. 입력·흐름 구조화   기록: 산출물 2·3번 (직접 입력 판단값은 5번)

```text
[방금 사례 기준 정리]
   · "방금 따라간 사례에서, 이 일을 시작하게 된 신호·계기는 뭐였어요?"
   · "'이 정도면 끝났다' 한 시점은요?"
   · "그때 쓴 자료·정보는 앞서 말한 것 외에 또 뭐가 있었어요? (슬랙·이메일·회의·세일즈맵 노트도)"
   · (자료마다) "실제로 본 부분은 전체/일부? 자주 비거나 변동되나요? 쓰기 전 확인하는 거 있어요?"
   · "본인이 직접 점수·등급·가능성·상태를 시스템/시트에 입력하는 게 있어요?"

[부분 요약 확인] "제가 이해한 흐름은 A→B→C, B에서 X 자료의 Y를 보시는 걸로 이해했어요. 맞나요?"
```

- **다음** — 시작·완료 조건 + 입력 자료 + 흐름 요약이 정리되면 5단계

---

### 5. 판단 지점·기준 (★핵심)   기록: 산출물 5번 (안 풀리면 10번)

```text
[판단 지점 열기] "이 흐름 중에 '그냥 진행할까 / 멈출까 / 확인할까' 결정하는 순간이 있었나요? 떠오르는 순서대로 다."
   ├ 판단이 나오면 → 각 판단마다 묻기:
        · "어떻게 결정하셨어요? 그 결과 선택한 이유는요?"
        · "그때 무엇을 보고 결정하셨어요?"
        · "반대로 처리한 사례 있었나요? 뭐가 달랐어요?"
        · "이 조건이 조금 달라지면 결과도 달라지나요?"
        · "다른 사람이 같은 상황 보면 어떻게 판단해야 하나요?"
   ├ "직감/케바케"라고 하면 → 프로토콜 B (비교 사례·경계값)
   └ 판단을 인지 못 하면 → "여기서 그냥 넘어가도 되는지 확인하신 게 있었나요?"

[기준 조건문 확인] "'X면 진행, Y면 보류, Z면 확인'으로 이해해도 될까요?"
   → 조건문화가 안 되면 산출물 10번 (기준 표현 미완성 / 암묵지·직감 영역)
```

- **다음** — 판단마다 [상황·선택 결과·참고 정보·기준·다르게 처리되는 경우] 핵심 잡히면 6단계 (빈 판단은 세션 후 raw 리뷰에서 미확인·보완으로)

---

### 6. 산출물·품질 조건   기록: 산출물 6·7번

```text
[산출물]
   · "이 업무 끝나면 무엇이 만들어지나요? 어떤 형태예요?"
   · "어디 저장되고, 누구한테, 어떻게 전달돼요?"
   · "정해진 양식·템플릿 있어요? 어디 있어요?"

[좋은 결과물 기준] "'이건 잘 만들었다' 느끼는 기준은요?"
   ├ 명확한 기준 있음 → "통과 기준이 정해져 있어요? (오타 0개 vs 의미 전달되면 OK) / 누가·언제·어떻게 확인해요? 안 맞으면?"
   └ 기준 모호 → 프로토콜 C ("효율적으로"·"잘" 등)
```

- **다음** — 산출물 형태·위치·전달·품질 기준 나오면 7단계

---

### 7. 주의·실수·예외·애매 (★핵심)   기록: 산출물 8번 (권한은 [권한·정책] 태그→11번)

```text
[열기]
   · "이 업무 하실 때 '이건 진짜 조심해야 한다'·'여기 잘 봐야 한다' 느끼는 부분 있어요?"
   · "정상 흐름대로 안 풀리는 상황 있어요? (입력 빠짐 / 기준 충돌 / 담당자 부재 / 형식 다름 / 수신자 반려)"
   ├ 사례 나오면 → 영역별로 묻기:
        · (주의) "놓치면 큰 문제 생기는 부분? 왜?"
        · (실수) "본인·다른 사람이 자주 하는 실수?"
        · (예외) "평소와 다르게 처리한 가장 최근 사례?"
        · (애매) "'어떻게 해야 할지 모르겠다' 느낀 적?"
        · "그런 상황 생기면 어떻게 처리하셨어요?"
   └ "다 설명했어요/별로 없어요" → 프로토콜 B (인지 부재 가능)

[권한·정책 — 의식적으로 묻기]
   · "권한이나 회사 정책 때문에 막혀서 다른 방법으로 돌아가신 적 있나요?"
   · "자료 가져오거나 전달·저장할 때 권한·보안·개인정보 제약 있어요?"
   → 있으면 발생 지점(3·4·6번)에 [권한·정책] 태그 → 11번 종합 (실제 부딪힌 것만)
```

- **다음** — 4영역 + 권한까지 짚으면 마무리(8단계)

---

### 8. 마무리 확인 (신청자와)

```text
[되돌려 확인] "제가 들은 걸 정리하면, 보통 [5요소 한 줄] 일을 하시는데, 흐름은 [3~4단계], 핵심 판단은 [1~2개], 자주 신경 쓰시는 건 [1~2개]예요. 맞아요?"   (형식지 전체는 보여주지 않음, 핵심만 1~2분)
   ├ 맞으면 → 종료
   └ 틀림/빠짐 → 즉시 정정(raw) / 빠진 건 미확인으로 / 많으면 후속 확인 약속
```

→ 신청자 인터랙션 종료. 아래는 진행자 정리.

---

## 세션 직후 점검 (진행자, 신청자 인터랙션 종료 후)

세션이 끝나자마자 진행자가 바로 확인할 것만:

- [ ] raw 데이터(녹음/로그/메모) 1개 이상 확보
- [ ] Phase 2 관련 발화가 있었다면 raw에 보존했는가
- [ ] 후속 확인 필요 시 별도 채널(DM 등) 약속

## 세션 후 — 미확인·보완 정리 (raw 리뷰 / AI 후처리)

미확인·보완 항목은 세션 중 즉석에서 파악하기 어렵고, **raw(녹음/로그)를 리뷰하면서 비로소 드러난다.** 후처리 프롬프트(프로토콜)가 raw→템플릿을 채우며 추론으로 못 채운 항목을 산출물 10번으로 정리한다.

- 유형: 자료 추가 확인 / 기준 표현 미완성 / 신청자 확인 필요 / 근거 부족 / 암묵지·직감 영역
- AX 매니저는 후처리 결과를 검토하며 빠진 미확인 항목을 추가한다.
- 보완 방법이 애매하면 신청자에게 별도 채널(DM)로 가볍게 확인 ("보완해주세요" 톤 X)

---

**전반적으로 막히면** → 프로토콜 B(막힘 대응표)를 두루 살펴 케이스 확인  ·  **모호어가 나오면** → 프로토콜 C(모호어 가이드)
(특정 답변에 대한 대응은 위 각 단계 코드블록 안 분기에 있음)
