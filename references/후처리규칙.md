# 후처리 규칙 ([후] 단계 — raw → 형식지) (references)

> [후] 단계가 이 규칙을 따라 `형식지템플릿.md` 11칸을 채운다.
> ※ 기존 인터뷰 프로토콜의 "AI 후처리 프롬프트"에서 *규칙·출력*만 추출·재구성. (스킬에선 "복사해 쓰는 프롬프트"가 아니라 [후] 단계가 따르는 규칙.)
> ※ draft. 실제 위치 = `references/후처리규칙.md` (SKILL.md와 같은 스킬 폴더).

## 역할

구현 방향을 정하기 전에, 신청자가 *현재 원업무를 실제로 수행하는 방식*을 충분한 깊이로 복원한다. 추출 깊이 기준 = "아무것도 모르는 후임자가 이 문서만 보고도 업무를 수행할 수 있는 수준". 구현 방식·자동화 방향은 여기서 단정하지 않는다(그건 Phase 2).

## 입력 (이 단계가 읽는 것)

- 인터뷰 raw (녹음 전사 / 세션 로그 / 메모)
- `구글폼응답_{신청자}.md` (세션 폴더)
- 첨부 자료 (양식·파일)
- `형식지템플릿.md` (채울 11칸 구조)

## 반드시 지킬 규칙

1. **raw 기반.** raw에서 합리적으로 추론 가능한 부분은 추론하되 끝에 **"(추론)"** 표시. 추론조차 어려우면 비우고 **10번**에 기록. raw와 무관한 내용 임의 생성 금지.
2. **신청자 발화 원본 표현 보존** (읽기 좋게 다듬되 의미 안 바꿈).
3. **못 채운 항목은 10번에 유형 분류**: 자료 추가 확인 / 기준 표현 미완성 / 신청자 확인 필요 / 근거 부족 / 암묵지·직감 영역. **`암묵지·직감 영역`은 끝까지 안 풀린 잔여에만** (마지막 안전망 — 남발 금지. *기준 표현 미완성·신청자 확인 필요*를 우선).
4. **주의어 검사** (LLM/API/MCP/자동화/효율화/최적화 등): (a) *원업무 사실·환경*이면 본문 보존 (예: "구글 드라이브에 저장"). (b) *이 업무를 AX화하는 구현 방법·방향 단정*이면 본문에 X → **11번 "Phase 2 참고 메모(비확정)"**로 분리. AI가 새로 판단한 구현 방식은 적지 않음.
5. **권한·정책·보안·PII 제약** → 발생 지점(3·4·6번)에 `[권한·정책]` 태그 + 11번 종합. *실제로 부딪힌 것만* (추정 X).
6. **신청자 직접 입력 판단값**(점수·등급·가능성·상태 등 직접 매겨 시스템·시트에 입력) → **5번**에 기록.
7. **전체 중 일부만 처리**하는 업무면, (a) 본인 적극 선별인지 (b) 원래 그 조건만 접수인지 구분해 1·2·3·5번 중 자연스러운 곳에. (b)면 조건 출처(사업부 원칙·업무 정의)도.
8. **8번 4영역**(주의·실수·예외·애매) 중 raw에서 못 찾은 영역은 비우지 말고 라벨: `[확인된 사례 없음]` / `[신청자 응답 어려움]` / `[추가 확인 필요]`.
9. **모호어**("적절히/중요하면/자연스럽게" 등)가 조건으로 안 풀린 채면 그대로 옮기지 말고 **10번(기준 표현 미완성)**.

## 출력

1. **형식지 1파일** — 채워진 템플릿 전체(11칸, **10번 미확인 포함**) + 문서 끝에 **★ 작성자 검토 필요 목록**(섹션 · 무엇이 불확실 · 왜). 파일명 `Phase1_형식지_{신청자}_{업무}.md` → **세션 폴더에 저장.**
2. **대화창 echo** — ★ 검토 필요 목록 + 10번 미확인 요약을 대화창에도 출력하고 **"이 내용은 형식지 파일 안에도 있어요"** 안내(누락 방지). 별도 파일 X.

## raw 수령 (3-way)

① 세션 폴더의 파일 ② 경로 붙여넣기(*다른 폴더에 있어도 OK*) ③ 텍스트 붙여넣기. **없으면**: "이 폴더에 raw를 넣거나 경로/내용을 알려주세요" 리마인드.

## Phase 2 인계

- **11번 = Phase 2(세션스킬) 입력** — 구현 의도·페인·권한 종합이 여기 모임.
